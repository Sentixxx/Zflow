## Key Decisions
- 采用 `router + handler + service + repository` 的后端结构，去掉无效的 `service` 空壳转发与 `usecase` 命名分叉；理由是减少重复抽象、降低耦合并保持行为不变。
- 删除 `internal/domain` 与 `internal/store` 兼容层，以及 legacy icon 回退路径；理由是仓库已无调用，保留只会增加维护成本。
- 收藏实现使用文章字段标记（`is_favorite`, `favorited_at`）而非独立文章表；理由是避免额外同步复杂度。
- 定时清理策略接入 `RefreshAllFeeds`，只清理非收藏过期文章，保留天数由 `settings/data` 控制；理由是简化调度入口并满足收藏保护诉求。
- 历史提交信息按 `.claude/commit_message_policy.md` 与 `memory_release.md` 重写为规范格式（含 Co-Authored-By）。
