# PLAN: RSS + LLM Reader (Phase 1)

## Milestones
0.5 数据层升级（SQLite）
- 迁移至SQLite持久化并启用WAL模式。
- 提供Feed/Folder/Entry CRUD接口。
- 接入Conditional GET（ETag/Last-Modified）。
- 启动15分钟后台刷新任务。

0. Go 后端基础
- 初始化 Go 服务结构与配置。
- 暴露订阅管理 API（创建/查询）。
- 完成首批 RSS/Atom 抓取链路与持久化。

1. 阅读器基础能力
- 订阅管理、文章列表、详情阅读、收藏、基础过滤。

2. 评分与分类管线
- 实现质量/相关性/新颖性评分。
- 接入文笔与反响信号（可用时）。
- 输出统一排序分与分类标签。

3. 探索机制
- 在主流中按比例随机掺入探索内容。
- 滑块调节（默认 1%，范围 0%-30%）。

4. 偏好档案与可解释性
- 用户可编辑偏好档案。
- 每条推荐展示分类与推荐依据。

5. 交付形态
- Web 端可用。
- PWA 支持离线安装。
- 打包应用可分发。

6. 隐私与可观测
- 默认零遥测。
- 本地统计面板（可关闭）。

## Scope
- In: RSS/Atom 阅读器、LLM 评分分类、探索注入、解释与偏好。
- Out: 非 RSS 来源深度集成、中心化 SaaS、商业化付费体系。

## Priorities
- P0: Go 后端可用性（订阅创建与首批抓取）。
- P0: 阅读器基础可用性。
- P0: 评分排序可用且可解释。
- P1: 探索比例可调与用户可控。
- P1: PWA 与打包分发。

## Risks & Dependencies
- LLM 评分成本与稳定性。
- 不同源质量差异导致推荐波动。
- 外部反响信号可获取性不稳定。
- 自托管部署复杂度影响使用门槛。

## Validation Strategy
- 手动验证：核心阅读流、探索比例、偏好编辑、推荐解释。
- 测试验证：评分聚合逻辑、排序稳定性、重复内容去重。
- 隐私验证：默认配置下无外发遥测请求。
