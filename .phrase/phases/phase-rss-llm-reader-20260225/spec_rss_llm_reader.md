# SPEC: RSS + LLM Reader (Phase 1)

## Summary
构建一个开源、自托管 RSS 阅读器（RSS/Atom only），提供 LLM 驱动的内容评分、分类与探索内容注入，以降低信息过载并缓解信息牢房。

## Goals
- 提供可用的基础阅读器能力：订阅、阅读、收藏、过滤、分类。
- 提供多维评分：质量、相关性、新颖性（辅以文笔、反响信号）。
- 提供可控探索机制：默认 1%，用户可调、可关闭。
- 提供可解释推荐：展示分类与推荐依据。
- 提供偏好档案：允许用户手动编辑并参与推荐提示。
- 交付形态覆盖 Web + PWA + 打包应用。
- 后端统一使用 Go 实现 API、抓取与调度能力。
- 默认零遥测，不进行集中数据采集。

## Non-goals
- 首版不接入非 RSS/Atom 来源（如平台 API 深度集成）。
- 首版不做中心化 SaaS 托管服务。
- 首版不以商业化指标作为成功标准。

## Technical Constraints
- 后端语言固定为 Go。
- 前后端通过 HTTP API 解耦，前端不得直接依赖抓取逻辑。

## User Flows
1. 导入订阅源
- 操作：用户添加 RSS/Atom URL。
- 反馈：成功抓取并显示订阅内容。
- 回退：抓取失败时提供错误信息与重试。

2. 阅读与筛选
- 操作：用户浏览主信息流，按分类/标签/分数过滤。
- 反馈：高价值内容优先展示，低价值内容可降权。
- 回退：用户可切换为时间排序。

3. 探索比例调节
- 操作：用户通过滑块调节探索比例（默认 1%）。
- 反馈：主流中随机掺入跨偏好内容。
- 回退：用户可设置为 0% 完全关闭。

4. 偏好档案编辑
- 操作：用户修改偏好关键词/主题倾向。
- 反馈：后续评分与推荐受新偏好影响。
- 回退：支持重置为默认偏好。

5. 推荐解释
- 操作：用户查看某条内容“为什么推荐”。
- 反馈：展示分类、核心维度与触发信号。
- 回退：用户可标记“减少此类内容”。

## Edge Cases
- 源内容质量极差导致评分失真。
- 外部反响信号不可用时的评分降级策略。
- 高频重复内容跨多个源重复出现。
- 用户偏好极端收窄导致探索内容体验下降。
- 大量订阅下抓取/排序性能退化。

## Acceptance Criteria
- 可成功添加并读取 RSS/Atom 源。
- 主信息流支持排序、分类、收藏和基础过滤。
- 每条内容可输出评分与分类。
- 探索比例滑块可实时生效，支持设置为 0%。
- 用户可编辑偏好档案并影响推荐结果。
- 推荐项可展示“推荐依据”。
- Web 与 PWA 均可运行；打包应用可安装并打开。
- 默认配置下无集中遥测上报。

## Success Signals
- 社区指标：GitHub Stars、Issues、PRs。
- 本地验证：实例内本地统计（可关闭）与用户主观满意度。
