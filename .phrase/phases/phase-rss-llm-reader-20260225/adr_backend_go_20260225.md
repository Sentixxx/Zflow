# ADR: 后端语言采用 Go

## Context
- 项目需要一个可自托管后端，承担 RSS/Atom 抓取、评分管线编排、推荐接口与任务调度。
- 目标是先快速交付可运行的 `task001`（订阅创建与首批抓取），并为后续评分与推荐保留并发与可维护性空间。

## Decision
- 后端语言固定采用 Go。
- 后端对前端暴露 HTTP API，前端仅通过 API 访问数据与触发操作。

## Alternatives
- Node.js: 生态完整但在高并发抓取与长期后台任务组织上不如 Go 直接。
- Python: 原型效率高，但首版目标不是算法实验而是稳定服务化。

## Consequences
- 需要在仓库维护 Go module 与 Go 测试链路。
- 后续 LLM 与外部信号接入统一放在 Go 服务层，避免前端耦合。
- 文档与任务按 Go 后端约束推进。

## Rollback
- 若未来出现团队维护瓶颈，可在新 phase 通过 ADR 再次评估语言迁移；当前 phase 不允许混用后端语言。
